# 🥗 Ecossistema de Inteligência Alimentar - São Luís/MA

## 📌 Sobre o Projeto
#### *Este software simula uma ferramenta de apoio à gestão de segurança alimentar na capital maranhense. O foco é centralizar dados de famílias em situação de vulnerabilidade, especialmente em áreas de periferia, palafitas e ocupações irregulares, como as comunidades ribeirinhas do Rio Anil e Bacanga.*

#### *O sistema visa resolver o problema de dados dispersos entre secretarias e a falta de análises rápidas, permitindo que a prefeitura se antecipe a crises sazonais (como enchentes entre janeiro e junho).*



## 🛠️ Justificativa das Estruturas de Dados
#### Seguindo os requisitos da UA I e o plano de ensino, utilizamos estruturas compostas para otimizar o processamento:

- **Dicionários:** Utilizados para o cadastro unificado de famílias (Chave: CPF/NIS), permitindo acesso de complexidade $O(1)$ aos atributos como renda, bairro e nível de insegurança.
- **Conjuntos (Sets):** Aplicados no monitoramento da cobertura territorial. Através da operação de diferença, identificamos instantaneamente quais bairros oficiais do IBGE ainda não receberam assistência municipal.
- **Tuplas:** Empregadas para o registro imutável de entregas (Data, Bairro, Quantidade), garantindo a integridade do histórico para auditoria.
- **Matrizes (Listas de Listas):** Estrutura base para o cruzamento sazonal de dados (Bairros x Meses), essencial para a geração do Heatmap de vulnerabilidade.

## 📊 Visualização e Inteligência (Parte 3)

1. Heatmaps: Para visualizar onde a fome é mais crítica em tempo real através de matrizes de risco.
2. Gráficos de Linha: Para antecipar a necessidade de estoque cruzando entregas e períodos chuvosos.
3. Gráficos de Pizza: Para ilustrar a distribuição dos perfis familiares atendidos (vulnerabilidade leve, moderada ou grave).
